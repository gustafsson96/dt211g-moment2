Arbetsprocess

Utveckling

Denna uppgift har skapats i programmet VS Code och kontinuerligt versionshanterats till GitHub. Arbetet har skett i två branches: main och development. Funktionalitet har inledningsvis skapats i development-branchen och färdiga lösningar har sedan sammanslagits med main-branchen via git merge. Arbetsfilerna har skapats i src-katalogen.

Ett antal filer har exkluderats från versionshantering via en .gitignore-fil. "node_modules"-katalogen och de optimerade publiceringsfilerna i "dist"-katalogen är två exempel.

Verktyg

Uppgiften skapades i NodeJS som ett npm-projekt, inledningsvis med kommandot npm init. För att automatisera delar av utvecklingsarbetet installerades paketet Parcel som en dev dependecy (för att uppfylla krav på möjlighet för återställning via npm install).

Bilder har inkluderats i jpg-format för att sedan genomgå optimering och konvertering till AVIF-format via "as"-parametern i kombination med en minskad bredd på 600. Denna funktionalitet har implementerats utefter den officiella dokumentationen för Parcel (länk till relevant dokumentationssida) med hjälp av picture-taggen i HTML-koden. En sharp.config.json-fil har skapats som styr kvalitén på filformaten (avif = 50 och jpg = 80).

För att säkerställa bakåtkompabilitet i webbläsare har "browserlist" inkluderats i package.json, även detta utefter guidning av Parcel-dokumentationen (länk till relevant dokumentationssida).

Den färdiga webbplatsen har förberetts för publicering genom att kommandot npm run build (genväg för parcel build) har körts i terminalen. Parcel har då optimerat de kopplade HTML, CSS och JavaScript-filerna och lagt dessa i "dist"-katalogen för publicering.

Publicering

Uppgiften har publicerats på Netlify via main-branchen för GitHub-repot. På så sätt har automatiserad publicering skett då jag har förgrenat commits från development-branchen till main-branchen via git merge för att sedan skicka uppdateringarna till uppgiftens remote repository på GitHub via git push.
